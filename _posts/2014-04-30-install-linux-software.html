安装Linux软件

开放源码，编译程序与可执行文件

  • 源代码:使用文本编辑器编写的程序代码，纯文本文件。
  • 可执行文件:二进制可执行程序，是计算机唯一可以识别的形式。
  • 编译程序:将源代码文件编译成为可执行的二进制文件。

$ file yourfile #查看文件的属性(directory,excutable,text and so on)

函数库

  • 函数库:类似子程序的角色，可以用来被调用来执行的一段功能函数。
  • 动态函数库:扩展名为.so。编译时，动态函数库的内容不被整合到可执行文件当中，而是当可执行文件要使用到函数库的机制时才会去读取函数。使用动态函数库编译出来的程序，不可独立执行。
  • 静态函数库:扩展名为.a。编译时，静态函数库会直接整合到可执行程序当中。可独立执行。

make 与 configure

configure或config：软件开发商编写的，用来检测用户的操作环境，以及该操作环境是否有软件开发商需要的其他功能。该检测程序执行完毕以后，会生成一个名为Makefile的规则文件。

make:根据检测程序（config或者configure）生成的Makefile文件，对软件的所有源代码文件进行编译。

通常，检测程序会做的工作主要有一下几个方面：

  • 是否有合适的编译程序可以编译本软件的程序代码
  • 是否已经存在本软件所需的函数库或其他的相关软件
  • 操作系统是否适合本软件，包括操作系统的内核版本
  • 内核的头定义文件（header include）是否存在（驱动程序必须要的检测）

Tarball软件

Tallball软件：:将源代码文件以tar打包，然后再以压缩技术来压缩而成的文件。

Tarball是一个软件包，解压之后主要包括以下内容：

  • 源代码文件
  • 检测程序文件（可能是configure或者config等文件名）
  • 本软件的简易说明与安装说明（INSTALL或者README）

安装与升级软件

  • 以源码通过编译来安装与升级(Tarball安装）
  • 以编译好的二进制程序来安装与升级（Red Hat系统的RPM软件管理机制与yum在线更新模式，Debian使用的dpkg软件管理机制与APT在线更新模式等）

Tarball软件安装的基本流程

  • 下载Tarball文件
  • 解压缩Tarball文件，生成源代码文件
  • gcc进行源码编译
  • gcc进行函数库，主程序，子程序的链接，以形成主要的二进制文件
  • 将上述的二进制文件以及相关的配置文件安装至自己的主机上面
