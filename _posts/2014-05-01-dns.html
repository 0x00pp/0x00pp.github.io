DNS原理

与DNS相关的名词解释

  • DNS:Domain Name System,域名系统。
  • hostname:主机名。
  • domain name:域名。
  • BIND:Berkeley Internet Name Domain，实现DNS协议的一套软件。特点：阶层式管理主机名对应IP的系统。
  • FQDN:Full Qualified Domain Name，由hostname与domain name组成。

For example:主机名为www的有：www.google.com,www.seednet.net,www.hebut.edu.cn等等。

与DNS相关的配置文件

  • /etc/hosts:主机名与IP对应的记录文件
  • /etc/resolv.conf:DNS服务器地址

DNS的分层架构

  • TLDs:Top Level Domains,顶级域名。
  • gTLD:Generic TLDs,一般顶级域名，例如.com,.org,.gov等。
  • ccTLD:Country Code TLDs,地区顶级域名，例如.uk,.jp,.cn等。
名称 代表意义
com 公司，行号，企业
org 组织，机构
edu 教育单位
gov 政府单位
net 网络，通信
mil 军事单位

通过DNS查询主机名IP的流程

以 www.hebut.edu.cn为例进行解释

  1. 收到用户请求,先查看本身有没有记录，若武则向.(root)查询。
  2. 向顶层.(root)查询。
  3. 向第二层的.cn服务器查询。
  4. 向第三层的.edu.cn服务器查询。
  5. 向第四层.hebut.edu.cn服务器查询
  6. 记录缓存并回报用户

分层DNS的优势

  • 主机名的修改仅需更改自己的DNS即可，不需要通知其他人。
  • DNS服务器对主机名解析结果的缓存时间。
  • 可持续向下授权。
