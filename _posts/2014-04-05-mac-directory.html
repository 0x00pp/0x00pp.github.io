Mac OS X's Directory

Finder中看到的目录结构

Finder是Mac OS X的资源管理器，是一个GUI(graphical user interface)程序。相对于CLI(command line interface)，GUI具有易操作性，对用户友好等优势。
Finder中看到的目录结构如下：

  • Applications:OS X的应用程序的文件夹。
  • System:OS X系统文件文件夹。该目录下只有一个Library文件夹。
  • Library:OS X系统资源文件夹。
  • Users:OS X非root用户文件夹。OS X是一个多用户的操作系统。每一个登录OS X的非root用户，都会在Users下有自己的用来存储数据的文件夹。
  • 用户信息:一些简单的用户信息。
  • opt:我个人因为装过一个软件，所以产生了这个目录。这也说明，在Finder中看到的目录不是固定的，可能会因为安装一些软件，系统自动生成一些目录。

Terminal中看到的目录结构

OS X使用的是Unix文件目录结构。由根目录/衍生出其他的目录，即根目录/下挂载了上面提到的/Applications,/System,/Library,/Users等等重要文件夹。因为对于普通用户来说，没有必要接触系统底层的一些缘故，所以OS X默认隐藏了很多文件，而这些隐藏的文件夹在Terminal下，是可以完整看到的。

  • /Applications:
  • /System:
  • /Library:
  • /Users:
  • /bin:基本的命令行命令。
  • /sbin:进阶的命令行命令。
  • /etc:系统文档。
  • /var:频繁改动的文档。
  • /usr:Unix软件资源。
